Executive Summary

This penetration test focused on identifying vulnerabilities within a controlled lab environment simulating real-world systems. Key findings included outdated software, weak encryption algorithms, and misconfigurations. Exploitations demonstrated unauthorized access via known vulnerabilities, culminating in potential data exfiltration and persistent access using custom payloads.

Mitigation strategies include regular patching, upgrading end-of-life operating systems, restricting SSH/SMB services, and using encryption best practices. By following these recommendations, organizations can reduce the attack surface and strengthen their security posture.

Passive OSINT

Summary: Publicly available data for “Discount Tire” (selected company) was collected using MXtoolbox, Whois, LinkedIn, Sublist3r and others. Domain records, emails, subdomains, and document metadata were identified.

Recommendations: Use social media carefully; limit personal/company data exposure. Avoid posting sensitive internal information on public websites. Regularly review privacy settings and corporate policies for data leaks.

Automated Information Recon

Summary: Tools such as SpiderFoot, DNSenum, Metagoofil, and Sherlock were employed to gather hidden OSINT artifacts and metadata from public resources.

Recommendations: Periodically audit and remove old or sensitive data from public-facing sites. Use crawlers to self-assess possible exposures. Review social media and online service accounts for username exposures.

Nmap Scan

Summary: Nmap scans were conducted on two VM targets to identify open ports, services, and OS fingerprinting. Many outdated services (FTP, HTTP, MySQL) were found, along with legacy Windows OS on one host.

Recommendations: Upgrade Apache from version 2.2.8. Restrict SSH (port 22) to trusted IPs. For Host 192.168.0.3, upgrade from Windows XP/Server 2003 to a supported OS. Disable SMB ports (139/445) when not needed.

OpenVAS Vulnerability Assessment

Summary: OpenVAS scans uncovered outdated encryption protocols, end-of-life operating systems, and clear-text transmissions.

Remediation: Upgrade or replace EOL OS. Reconfigure weak SSH algorithms. Enforce HTTPS instead of HTTP. Disable TCP timestamps and ICMP timestamp replies.
